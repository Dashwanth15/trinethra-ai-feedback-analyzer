## Overview

Trinethra turns raw supervisor session transcripts into structured, reviewable feedback reports — locally, privately, and without any external API calls.

Paste a transcript. Get a scored analysis with verbatim evidence, coverage gaps, and follow-up questions — all validated before reaching the UI. The AI assists; the human decides.

---

## Features

- **Structured 9-field output** — score, summary, strengths, weaknesses, evidence, gaps, follow-up questions, reasoning, confidence
- **Verbatim evidence verification** — every extracted quote is cross-referenced against the source transcript
- **Gap analysis** — flags evaluation dimensions not addressed in the session
- **Deterministic confidence scoring** — computed from transcript length and evidence density, not generated by the LLM
- **KPI allowlist enforcement** — extracted KPIs validated against a fixed controlled vocabulary
- **Retry pipeline** — up to 3 retries with escalating prompt strictness on parse or validation failure
- **Fully local inference** — no data leaves the machine, no API keys, offline-capable

---

## Hallucination Guardrails

- **Evidence verification** — quotes must exist verbatim in the source transcript or are rejected
- **JSON-only prompting** — system prompt enforces raw JSON output, no freeform text
- **Schema validation** — Pydantic rejects malformed, incomplete, or out-of-range responses
- **KPI allowlist** — 8 allowed terms; anything outside the set is flagged
- **Deterministic confidence** — `High / Moderate / Low` computed in Python, never delegated to the model
- **Retry with escalation** — failed parses trigger stricter prompt templates, not silent fallbacks

## Local AI Workflow

This project runs entirely on-device via [Ollama](https://ollama.com/).

**Why local?**
- Transcript data stays private — nothing is transmitted externally
- No API keys, no usage costs, no rate limits
- Works offline in restricted network environments
- Reproducible behavior across runs with a fixed model version

Prompts are optimized for `phi3:mini` — flat JSON schema, minimal nesting, explicit field instructions — to maximize output reliability on consumer hardware.

## Setup

### Prerequisites

- [Ollama](https://ollama.com/) installed and running
- `phi3:mini` pulled: `ollama pull phi3:mini`
- Python 3.10+ and Node.js 18+

### Backend

```bash
cd backend
python -m venv venv && source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

App → `http://localhost:5173` · API → `http://localhost:8000` · Docs → `http://localhost:8000/docs`
